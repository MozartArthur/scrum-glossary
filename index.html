Диаграмма сгорания задач

Выполнил Стрижов Егор

Диаграмма сгорания задач — диаграмма, показывающая количество сделанной и оставшейся работы. Является частью фреймворка Scrum. Обновляется ежедневно с тем, чтобы в простой форме показать подвижки в работе над спринтом. График должен быть общедоступен. Существуют разные виды диаграммы: диаграмма сгорания работ для спринта, диаграмма сгорания работ для выпуска проекта.

Scrum-meeting

Выполнил Косоруков Роман

Daily Scrum Meeting - это ежедневные "совещания" участников команды, работающей над одним проектом, которые проводятся с целью выявления проблем в работе.

Спринт

Выполнил Гунько Виктор

Спринт — итерация в Scrum, в ходе которой создается инкремент бизнес-продукта. Жестко фиксирован по времени. Длительность одного спринта: 1-4 нед. Чем короче спринт, тем более гибким является процесс разработки. С другой стороны, при более длительных спринтах скрам-команда уменьшает издержки на совещания, демонстрации продукта и т. п.

Planning poker

Выполнил Иванов Дмитрий

Покер планирования — техника оценки, основанная на достижении договорённости, главным образом используемая для оценки сложности предстоящей работы или относительного объёма решаемых задач при разработке программного обеспечения.

DoD

Выполнила Кривцун Марина

Definition of Done (DoD) – критерий, определяющий степень готовности задачи. Применяется в тех случаях когда окончательно невозможно проверить готовность задачи, например, если элемент функционала находится в другой скрам команде или компании. Описание DoD начинается со строчки «done = », например, done = функционал реализован в тестовой среде, требуется выгрузка и проверка в основной среде.
